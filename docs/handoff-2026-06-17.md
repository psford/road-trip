## How to work (non-negotiable, given how this session went)
1. **Verify before claiming.** No "done / works / good" about UI without it **seen running on a
   simulator** (screenshot). Passing `xcodebuild test` is necessary, not sufficient — it does not
   prove a screen renders. (`show-dont-claim-app-works`.)
2. **The container can't run a simulator** (Linux, no Xcode). Blind UI authoring is what produced
   the facade. So: do the **non-visual engine** here (API client, upload, EXIF — testable); the
   **visual UI** needs a build→run→see loop — either the native-Mac Claude (Xcode bridge, can see
   the sim) or Patrick running each increment. (`topology-container-vs-native-claude`.)
3. **Don't deflect hard asks to easier ones.** Object on merits (won't work / bad idea / downsides),
   never on effort. (`dont-deflect-hard-to-easy`.)
