# iOS Offline Shell — Phase 5: Loader integration + boot routing

**Goal:** Replace `src/bootstrap/loader.js` end-to-end with the orchestrator that wires Phases 1–4 together (CachedFetch + TripStorage + FetchAndSwap + Intercept). The new loader is the only entry point for the iOS shell. Also: ship the AC3.6 fallback retry/back UI, the AC2.4 glasses indicator on the home page, and the bootstrap-time `tripStorage.js` shell copy needed for offline-first booting.

**Architecture:** Tiny IIFE in `src/bootstrap/loader.js` that (1) sets `platform-ios` before first paint, (2) monkey-patches `FetchAndSwap.fetchAndSwap` to re-inject `<link rel="stylesheet" href="/ios.css" data-ios-css>` after every swap (since `document.head.innerHTML` gets replaced on each swap), (3) reads `TripStorage.getDefaultTrip()` and document-swaps to its postUrl (or `/` if null), (4) installs `Intercept.installIntercept()` once after the first swap, (5) catches any error and renders `fallback.html` with a wired-up retry button. `index.html` is updated to load all five bootstrap modules with `defer` so they execute in document order. The `tripStorage.js` shell copy is committed (regenerated by an npm script) so the loader has TripStorage available before any network call.

**Tech Stack:** Vanilla JS (ES2020). Vitest + jsdom + fake-indexeddb for unit tests. Capacitor sync + iOS Simulator for the manual smoke.

**Scope:** Phase 5 of 8 from the iOS Offline Shell design.

**Codebase verified:** 2026-04-19.

## Bootstrap dependency decision (recorded for future reference)

The loader needs `TripStorage` available at boot to call `getDefaultTrip()`. `tripStorage.js` lives in `src/RoadTripMap/wwwroot/js/` (App Service). The iOS shell runs at `capacitor://localhost` and ships only files in `src/bootstrap/`. Three options were considered:

1. **Commit `src/bootstrap/tripStorage.js` as a copy.** ← chosen.
2. Fetch `tripStorage.js` from App Service via `CachedFetch` at boot, then `eval` the source.
3. Inline a minimal `getDefaultTrip` subset into the loader.

Option 1 keeps the loader pure orchestration, has zero boot-time network dependency, and matches an existing pattern (`wwwroot/bundle/*` is a committed generated artifact per CLAUDE.md). Drift risk is addressed by the `npm run prepare:ios-shell` sync script (Task 1) and the CLAUDE.md note.

## Module contract

`src/bootstrap/loader.js` (full source given in Task 3). Single IIFE, no exposed globals (the loader is purely procedural). Wraps `FetchAndSwap.fetchAndSwap` to add ios.css re-injection. Boot routing reads `TripStorage.getDefaultTrip()`. Top-level try/catch falls back to `fallback.html`.

`src/bootstrap/index.html` script tags (in this order, all `defer`): `cachedFetch.js`, `tripStorage.js`, `fetchAndSwap.js`, `intercept.js`, `loader.js`. `defer` ensures execution in document order after parsing — loader.js runs after all four dependencies are installed.

### Task 3: New `loader.js` — boot routing + ios.css injection + error fallback

**Verifies:** `ios-offline-shell.AC2.1`, `ios-offline-shell.AC2.2`, `ios-offline-shell.AC3.6`.

**Files:**
- Replace: `src/bootstrap/loader.js` — full rewrite. The 239-line bundle-loading IIFE is replaced.

**Implementation:**

Full file contents (verbatim):

```js
(async function bootstrap() {
    try {
        // Set platform-ios class before first paint (preserved from previous loader; AC10.1).
        document.body.classList.add('platform-ios');

        // Wrap FetchAndSwap.fetchAndSwap so every swap re-injects ios.css.
        // (document.head.innerHTML is replaced on every swap, removing any prior <link>.)
        // Note: the wrapper kicks in only AFTER loader.js has run. Phase 3/4 unit tests
        // do NOT exercise this path because they call FetchAndSwap.fetchAndSwap directly
        // before any loader.js eval. The "ios.css applied to every fetched page" intent
        // is verified by Phase 5 Task 5's bootstrap-loader tests + Phase 7's on-device matrix.
        if (FetchAndSwap && typeof FetchAndSwap.fetchAndSwap === 'function') {
            const original = FetchAndSwap.fetchAndSwap;
            FetchAndSwap.fetchAndSwap = async function (url, options) {
                await original(url, options);
                _ensureIosCss();
            };
        } else {
            throw new Error('Bootstrap: FetchAndSwap is not loaded');
        }

        if (typeof TripStorage === 'undefined' || typeof TripStorage.getDefaultTrip !== 'function') {
            throw new Error('Bootstrap: TripStorage is not loaded');
        }

        // Boot routing (AC2.1, AC2.2)
        const defaultTrip = TripStorage.getDefaultTrip();
        const bootUrl = defaultTrip ? defaultTrip.postUrl : '/';
        await FetchAndSwap.fetchAndSwap(bootUrl);

        // Install delegated nav AFTER first swap (the swapped page is what carries
        // the trip / home links; intercept fires on user clicks from now on).
        if (Intercept && typeof Intercept.installIntercept === 'function') {
            Intercept.installIntercept();
        }

        // Remove the bootstrap-progress shim now that the real page has rendered.
        const progress = document.getElementById('bootstrap-progress');
        if (progress) progress.remove();
    } catch (err) {
        console.error('Bootstrap failed:', err);
        await _renderFallback(err);
    }

    function _ensureIosCss() {
        if (!document.head.querySelector('link[data-ios-css]')) {
            const link = document.createElement('link');
            link.rel = 'stylesheet';
            link.href = '/ios.css';
            link.setAttribute('data-ios-css', 'true');
            document.head.appendChild(link);
        }
    }

    async function _renderFallback(_originalError) {
        // AC3.6: offline + cache miss → fallback.html with retry.
        try {
            const res = await fetch('fallback.html');
            const html = await res.text();
            document.body.innerHTML = html;
            const retry = document.getElementById('bootstrap-retry');
            if (retry) retry.addEventListener('click', () => location.reload());
        } catch {
            document.body.innerHTML =
                '<div style="padding:2rem;font-family:system-ui">' +
                '<h1>Unable to load</h1>' +
                '<p>Tap anywhere to retry.</p>' +
                '</div>';
            document.body.addEventListener('click', () => location.reload(), { once: true });
        }
    }
})();
```

Notes for executor:
- `defer` on the script tags guarantees `cachedFetch.js`, `tripStorage.js`, `fetchAndSwap.js`, `intercept.js` have all run before this IIFE starts.
- `document.body.classList.add('platform-ios')` runs before any `await` (AC10.1).
- The wrapper around `FetchAndSwap.fetchAndSwap` is intentionally a runtime monkey-patch rather than introducing an `onAfterSwap` API. It keeps Phase 3/4 unchanged — the iOS-specific `ensureIosCss` lives only in the loader.
- AC3.6 expects a `#bootstrap-retry` element in fallback.html; Task 4 adds it.

**Verification:**

- Tests in Task 5; smoke in Task 6.
