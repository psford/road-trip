# Human Test Plan: Map POI Feature

## Prerequisites
- Application running locally against a database with seeded POI data (run the POI seed script against dev database)
- All automated tests passing: `dotnet test tests/RoadTripMap.Tests/`
- At least one trip created with 2+ geotagged photos at known locations
- Browser with DevTools accessible (Chrome or Firefox)
- Seeded POI data includes all 5 categories in the Yellowstone/Grand Teton area (lat ~44, lng ~-110) for concentrated testing

## Phase 1: POI Marker Rendering

| Step | Action | Expected |
|------|--------|----------|
| 1.1 | Open the post page for an existing trip. Click the pin-drop map button to open the map. | Map opens in a modal or full-screen view. |
| 1.2 | Navigate to the Yellowstone area (lat ~44.4, lng ~-110.8). Zoom to level 10 or higher. | Colored circle markers appear on the map representing POIs. Labels appear next to markers. |
| 1.3 | Pan the map to an ocean area with no POIs (e.g., mid-Atlantic). | All POI markers disappear. No orphaned markers remain. |
| 1.4 | Open a trip view page (not post page). Navigate to the same Yellowstone area at zoom 10+. | Same colored circle markers appear with labels, identical to the post page rendering. |
| 1.5 | With POIs visible, identify markers from each category and compare colors against the spec: National Park = dark green (#2d6a4f), State Park = light green (#52b788), Natural Feature = brown (#7b2d26), Historic Site = purple (#6c4ab6), Tourism = gold (#d4a017). | Five visually distinct colors are present. Each category is distinguishable at a glance without needing to read labels. |

## Phase 2: Park Polygon Restyling

| Step | Action | Expected |
|------|--------|----------|
| 2.1 | On any map page, navigate to Franconia Notch State Park (lat 44.14, lng -71.68). Zoom to level 8-10. | Park polygon displays with a bold green fill that is noticeably more opaque/vivid than surrounding terrain. Dark green outline visible on park boundary edges. |
| 2.2 | Zoom out to zoom level 6. Look for the "Franconia Notch State Park" text label. | Label is visible on the map at zoom 6. |
| 2.3 | Zoom out to zoom level 5. | Label disappears (below minzoom threshold). |
| 2.4 | Zoom back in to level 8-12. | Label reappears and remains readable at all zoom levels within this range. |
| 2.5 | Open browser DevTools console. Run: `map.getStyle().layers.filter(l => l.id === 'park-labels-custom')` | Returns an array with one layer object (confirms the custom label layer exists). |
| 2.6 | Open a trip view page and navigate to the same Franconia Notch area. | Identical bold green styling and park labels visible, matching the post page appearance. |

## Phase 3: Tap-to-Pin Interaction

| Step | Action | Expected |
|------|--------|----------|
| 3.1 | Open the post page pin-drop map. Navigate to an area with visible POI markers at zoom 10+. Click/tap a POI marker. | A popup appears containing: the POI name, a green "Use this location" button, and a gray "Pick nearby spot" button. |
| 3.2 | Click outside the popup (on the map background). | Popup dismisses. No pin is placed. |
| 3.3 | Click the same POI marker again to reopen the popup. Click "Use this location". | Popup dismisses. A pin marker appears at the POI's location. Latitude and longitude form fields are populated with the POI's centroid coordinates. The place name field is filled with the POI's name. The upload/next step becomes enabled. |
| 3.4 | Dismiss/reset the pin. Click a different POI marker. Click "Pick nearby spot". | Popup dismisses. Map animates (flyTo) to zoom 13 centered on the POI. No pin is placed yet. |
| 3.5 | After the zoom animation completes, click/tap on a specific spot on the map near the POI. | A pin is placed at the clicked location. Latitude/longitude fields populate with the clicked coordinates. Geocoding populates the place name. Normal pin-drop behavior works. |
| 3.6 | Open a trip view page. Navigate to an area with visible POI markers at zoom 10+. Click/tap a POI marker. | NO popup appears. No "Use this location" or "Pick nearby spot" buttons. The click does nothing to the POI marker. |
| 3.7 | On the same view page, click an existing photo marker (not a POI marker). | Photo marker popup works normally (shows photo info). POI layer has not broken existing photo marker functionality. |

## Phase 4: Zoom-Based Category Filtering (Visual)

| Step | Action | Expected |
|------|--------|----------|
| 4.1 | Open any map page. Zoom out to level 5 in an area with seeded POIs across all categories. | Only national park markers are visible. No state parks, natural features, historic sites, or tourism markers. |
| 4.2 | Zoom in to level 8. | State park and natural feature markers appear alongside national parks. Historic sites and tourism markers remain hidden. |
| 4.3 | Zoom in to level 10. | All category markers become visible including historic sites and tourism. |
| 4.4 | Open browser DevTools Network tab. Zoom/pan the map. Observe requests to `/api/poi`. | Requests include `zoom`, `minLat`, `maxLat`, `minLng`, `maxLng` query parameters. Response JSON arrays contain the expected categories for the current zoom level. |

## Phase 5: Pin-Drop Default Location

| Step | Action | Expected |
|------|--------|----------|
| 5.1 | Create a new trip with zero photos. Open the pin-drop UI. | Map opens showing the full continental United States. Zoom level is approximately 4. Center is approximately central Kansas (lat ~39.8, lng ~-98.6). |
| 5.2 | Post a photo pinned to Franconia Notch (lat 44.14, lng -71.68). Close the pin-drop UI. Reopen the pin-drop UI to add another photo. | Map opens centered near lat 44.14, lng -71.68 at approximately zoom 10. The surrounding geography shows the White Mountains / New Hampshire area. |
| 5.3 | Post a second photo pinned to the Grand Canyon (lat 36.1, lng -112.1). Close and reopen the pin-drop UI. | Map now centers on the Grand Canyon area (lat ~36.1, lng ~-112.1), not the previous Franconia Notch location. The last-posted photo's location is used. |
